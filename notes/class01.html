HTML Basics

HTML - History of HTML

• HTML => HyperText Markup Language

• Invented in 1989 by Sir Timothy Berners-Lee

• EVERY website you've ever visited is rendered in HTML

• Web browsers parse (read) HTML and render (display) the results

HTML - Say what?

HT => HyperText

HyperText is text which contains links to other texts.

M => Markup

Markup describes a system of describing or annotating text files. Markup describes the structure/layout of a webpage.

L => Language

Just like any language, HTML has its own vocabulary, grammar, and syntax. You can think of HTML as the language that the web speaks.

HTML Structure: DOCTYPE

        <!DOCTYPE html>

• Short for "document type declaration"

• ALWAYS the first line of your HTML

• Tells the browser which "flavor" of HTML is being used

• All modern websites should use HTML5 DOCTYPE

HTML Structure: <html>

• <html> is HTML...

• All HTML should be contained inside <html></html>

• Represents the root of your HTML document

• Within our <html> tags, we have:

<head></head>

<body></body>

HTML Structure: <head>

• <head> : first tag inside <html></html>

• Adds additional, behind-the-scenes content

• Not displayed; but is machine parsable

• <title> : adds title to page

• <meta> : adds data...about data

• Some meta data: page description, keywords, author

• External CSS documents are linked in the <head>

HTML Structure: <body>

• <body> : second tag inside <html></html>

• Follows <head></head>

• Contains HTML/content that will be displayed to user

• Vast majority of HTML will be placed inside the <body>

HTML Structure: Basic Structure

        <!DOCTYPE html>
        <html>
          <head>
            <title>The Website Title</title>
          </head>
      
          <body>
            <h1>ALL HTML CONTENT HERE!</h1>
          </body>
        </html>

Working with CSS

Working with CSS

Three ways to apply CSS to HTML:

1) Inline CSS

2) Embed a <style> tag inside the <head>

3) Link to an external file using the <link> tag

Inline CSS

        <p style="color: blue; font-size: 14px;">
          What is Brooklyn's official motto?
        </p>
      
        <p style="color: blue; text-decoration: underline;">
           Een Draght Mackt Maght.
        </p>

• Not usually good practice

• Does not separate our concerns (HTML/CSS)

• Makes debugging difficult

• Use when writing HTML emails

• ...or maybe modifying a "plugin"

Do NOT Use Inline CSS!*

* Some notable exceptions

<head> embed

        <!DOCTYPE html>
        <html>
        <head>
          <title>Super Awesome Website</title>
          <style type="text/css">
            h1 {color: red;}
            p {color: blue; font-size: 14px}
          </style>
        </head>
        <body>
          ...

• Better than inline styles

• Bloats HTML pages

• Can make debugging large applications difficult

External CSS

       <!DOCTYPE html>
       <html>
       <head>
         <title>Super Awesome Website</title>
         <link rel="stylesheet" type="text/css" href="style.css">
       </head>
       <body>
        ...

• Like <title>, links to CSS files go in <head>

• Truly separates HTML from CSS

• Easiest to debug; just go to your CSS file!

External CSS

           <link rel="stylesheet" type="text/css" href="style.css">

• rel

⇒ Specifies the relationship between the current document and the linked document

• type

⇒ Specifies the media type of the linked document

• href

⇒ Specifies the location of the linked document

Classes and IDs

IDs - Embedding in HTML

• IDs are attributes

• Assigned to only one HTML element per page

• Identify the element for styling/scripting

• Cannot contain spaces


        <p id="important-title">Hello, World!</p>
IDs - Targeting with CSS

• To style the following ID:


        <p id="important-title">Hello, World!</p>

• Use # + the ID's name:


        #important-title {
          color: red;
        }

• CodePen Exercise

Classes - Embedding in HTML

• Classes are attributes

• Assigned to multiple HTML elements per app

• Identify the elements for styling/scripting

• Cannot contain spaces


        <p class="planet-greeting">Hello, World!</p>
        <p class="planet-greeting">Hello, Mars!</p>
        <p class="planet-greeting">Hello, Jupiter!</p>
Classes - Targeting with CSS

• To style the following classes:


        <p class="planet-greeting">Hello, World!</p>
        <p class="planet-greeting">Hello, Mars!</p>

• Use . + the class's name:


        .planet-greeting {
          color: pink;
        }

• This will select <p> tags with the class of planet-greeting


        p.planet-greeting {
          color: pink;
        }

•CodePen

IDs vs. Classes

• General rule:

⇒ Use an ID when you are styling ONE specific item

⇒ Use a CLASS when you are styling a GROUP of items

⇒ Or if starting your own fresh project...only use classes for style

• Note:

⇒ Elements cannot have more than one ID

⇒ Elements can have multiple classes

The Box Model: Padding

        // Method One:
        // targets each side individually
        p {
          padding-top: 5px;
          padding-bottom: 5px;
          padding-right: 5px;
          padding-left: 5px;
        }
      
        // Method Two:
          // starts at top, moves clockwise: right, bottom, left
        p {
          padding: 5px 5px 5px 5px;
        }
      
        // Method Three:
          // padding: (top/bottom) (left/right)
        p {
          padding: 5px 5px;
        }
      
        // Method Four:
        // will target all four sides (top, right, bottom, left)
        p {
          padding: 5px;
        }
The Box Model: Borders

• Borders consist of three properties:

⇒ border-width: a px value for width

⇒ border-style: the style (solid, dotted, dashed, double)

⇒ border-color: the color


        p {
          border: 2px solid black;
        }
      
        p {
          border: rgb(255, 0, 0) 2px dashed;
        }
The Box Model: Margin

• Follows same syntax as padding


        p {
          margin-top: 5px;
          margin-bottom: 5px;
          margin-right: 5px;
          margin-left: 5px
        }
      
        p {
          margin: 5px 5px 5px 5px;
        }
      
        p {
          margin: 5px 5px;
        }
      
        p {
          margin: 5px;
        }

The Display Property

The Display Property

• Two possible display values for HTML elements are*:

⇒ block

⇒ inline

• Controlled by the CSS property: display

⇒ display: block;

⇒ display: inline;

• Referred to as either block or inline elements

*We'll talk about addl. display values later

Block Elements

• Take up 100% of the width of their parent

• Can have margin and padding assigned

• Can set height and width

• Placed below previous elements in the markup; break to a new line

Block Elements
Block Elements

• Examples of block elements:

⇒ <h1> thru <h6>

⇒ <p>

⇒ <ul>

⇒ <li>

Inline Elements

• Only take up the width of their content

• Can have any padding (top, left, bottom, right) assigned

• Can only have left and right margin assigned

• CANNOT set height and width

• Do not send elements to the next line

Inline Elements
Inline Elements

• Examples of inline elements:

⇒ <a>

⇒ <strong>

⇒ <em>
