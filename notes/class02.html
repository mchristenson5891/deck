GitHub - Repository/Repo

• Basic element of GitHub

• Contains all of a project's files (all the code)

• One or more users can contribute to a single repository

• Repositories are either public or private

Clone

• Copies/clones a remote repo to your machine

• This copy/clone is called a local repo

• Changes to the local repo will not affect the remote

• Multiple users can clone the same repo:

Commit

• A snapshot of changes to a repo

• Think of it as "saving" your local changes

• Commits do not change the original, remote repo

• Contains a message describing the changes made

Push

• "Pushes" your commits (saved changes) to a remote repository

• Allows other developers to see your changes and "pull them down" to their own local repos

JavaScript Basics: Variables

JavaScript Basics: Variables

• Often, you will want to store, or keep track of, data in a program

• Declare using the var keyword (note: ES6 introduced let and const)

• Will store the piece of data in memory

• Use the variable's name to "call" it (retrieve it from memory):


    let humanScore = 2;
    let computerScore = 2;
  
    humanScore
    // 2
  
    humanScore + computerScore
    // 4
Variables: Naming Rules

• Take care when naming your variables

• Variable names should reflect the data they describe

• Other naming conventions:

⇒ Names should start with a lowercase letter

⇒ Cannot use spaces; use hyphens or underscores instead

⇒ Better yet, use camelCase:


    let currentTemp = 70;
    let targetTemp = 73;

Browser Events

Browser Events

• Events that users perform in the browser:

⇒ click

⇒ mouse over

⇒ mouse out

⇒ key down

⇒ key up

⇒ form submit

• We can use JavaScript to track and respond to events

• More Events Info

Browser Events

Apply events in three steps:

1) Use document.getElementById() (or, any other query selector) to select the element you want to apply the event to:


    document.getElementById('click-me')

2) Apply the event name to the selected element:


    document.getElementById('click-me').onclick

3) Set the above equal to the function that will run when the event is triggered


    document.getElementById('click-me').onclick = function () {
      // do something
    }
Browser Events: Click Event

• Use .onclick to add a click handler to an element

• Events can be handled with both anonymous and named functions:

• Named function:


    document.getElementById('some-el').onclick = doSomething;
  
    function doSomething () {
      // function body
    }

• Anonymous function:


    document.getElementById('some-el').onclick = function () {
      // function body
    }

What is an Object?

An object is an unordered list of data that is stored as a series of name-value pairs. Each item in the list is called a property (functions are called methods). Consider this simple object:


   let coderDude = {
    firstName: "Mike", 
    favLanguage: "JavaScript"
   }
What is an Object?

   let coderDude = {
    firstName: "Mike", 
    favLanguage: "JavaScript"
   }

Think of an object as a list that contains items, and each item (a property or a method) in the list is stored by a name-value pair.

The property names in the example above are firstName and favLanguage. The values are “Mike” and “Javascript.”
